parse

Available Since: TIE (1.1.6, 1.2.0) SUIT (1.3.0)

Translates tokens to statements using the enclosed string

Syntax string $suit->parse ( string $nodes, string $return, array [$config] )
Parameters
nodes

An array of nodes.

return

The template we want to translate.

config

An array of optional parameters. (Optional. 'escape' => $suit->config['parse']['escape'])

Return Values

If $config['preparse'] is false or not set, it returns translated string. Else, it returns an associative array with the keys 'return' and 'taken'.

Examples
Basic Usage
Template 
{Text} (Text) \{Text\}
Code <?php
function nodeupper($params)
{
    $params['case'] = strtoupper($params['case']);
    return $params;
    /*
    $params['case'] is the string in between the opening and closing tags of the node.
    $params['suit'] is the SUIT object that the parse function was called on.
    $params['var'] is the 'var' key on the node if it was provided.
    $params['nodes'] is the nodes used in the parse function. Used in $suit->section->loop.
    $params['escape'] is the escape string used in the parse function. Used in $suit->section->loop.
    */
}

function nodelower($params)
{
    $params['case'] = strtolower($params['case']);
    return $params;
}

$nodes = array
(
    '{' => array
    (
        'close' => '}',
        'function' => 'nodeupper'
        /*
        'class' => Class: The class on which to call the function.
        'skip' => Boolean: True if nodes should not be parsed inside of this node.
        'strip' => Boolean: True if the node does not actually transform the inside of the opening and closing strings,
        but merely strips the opening and closing strings. Used in $suit->section->condition.
        'ignore' => Boolean: True if this node should be ignored. Used in preparsing.
        */
    ),
    '(' => array
    (
        'close' => ')',
        'function' => 'nodelower',
    )
);
$config = array
(
    'escape' => '\\'
    /*
    'label' => String: Provide the entry's label if you want to debug the function.
    'preparse' => Boolean: Return a list of ranges that have been parsed along with the translated string in an
    associative array. Used in $suit->section->loop.
    'taken' => A list of ranges that should not have anything parsed in.
    */
);
$template = $suit->parse($nodes, $template); //Result: TEXT text {Text}. See <a href="http://www.suitframework.com/docs/Escaping">Escaping</a>
?>
Version Notes As Python allows you to send defined functions as parameters, send functions without quotes in the nodes. In addition, the class key is not used.
Changelog
  • 1.1.7 - $template moved from parameter 1 to parameter 4. Escaping will work with tags wrapped directly around replacement.
  • 1.2.1 - New escaping added. See Escaping.
  • 1.2.2 - Optional Config Parameters Added.
  • 1.3.0 - Added nodes for sane recursing. Label added for debug system. Now part of the SUIT class.
  • 1.3.3 - Rewrites for efficiency regarding removing trees. As a result, some template syntax errors that would normally break the entire page do not. $match parameter removed.
  • 1.3.4 - Nodes use functions instead of eval. Preparsing added. $escape and $label compacted into $config.
  • 1.3.5 - Node functions return array of parameters instead of the transformed case.
