Changelog


Summer of 2008: Development of a forum system begins.
October 10, 2008: The concept of SUIT begins based off of forum's templating system.

Pre-Version 1 Changes:

To be added...

February 17, 2009: 1.0.0
February 18, 2009: 1.0.1 - Users are not forced to make there template folders and files writable. Doing so for code is now not supported, thus the admin area only allowed the editing of content.
February 18, 2009: 1.0.2 - Some markup left over from the main site was removed. As before the cloning function was pasting the code of the old file in the code box, which no longer meant anything, that display was removed.
February 19, 2009: 1.0.3 - The name you can give a file was now filtered. More useless aesthetics and code were removed.
February 20, 2009: 1.0.4 - This version introduced the concept of glue. Instead of defining a template as the name of a particular content file, and then automatically associating it with a code file of the same name, now this third entity controls which files are associated, and the availability of a template is dependent of the glue files existence, not content.
February 23, 2009: 1.0.5 - The separate admin areas were merged into one. The way glue was set up did not work with Windows, and was changed.
February 25, 2009: 1.0.6 - Conditionals and Looping were implemented in this version. By running functions, it translated content to work like conditionals or loops respectively without any direct parsing.
February 25, 2009: 1.0.7 - Fixed the Code Admin area's ability to view, which was broken in the conditional/looping implementation.
February 26, 2009: 1.0.8 - Glue Modification and Edit and Continue were added for the admin area. Glue modification allows content to automatically associate itself with a code file instead of editing the glue. Although it is a shortcut, it is more convenient, especially in cases of renaming. Regex was escaped for Conditionals and Looping allowing it to work with more tokens. Search Highlighting was added.
February 27, 2009: 1.0.9 - Search Highlighting would now work with HTML characters. Further RegEx bugs were fixed. To be consistent, the parameters for the replace function were flipped.
February 28, 2009: 1.1.0 - The Debug Option was ported to TIE as opposed of being controlled by a construct parameter. register_globals disabling and magic_quotes_runtime disabling were also ported to TIE.
March 8, 2009: 1.1.1 - General Cleaning. parsePhrases renamed to parseLanguages. Redirect Message now not mandatory.
March 8, 2009: 1.1.2 - Packaging problem with Language Updater.
March 14, 2009: 1.1.3 - Replace now works with tokens inside themselves. PCRE now built recursively. Parse returns arrays as opposed to just translating the content. Pagination has a pages parameter, deciding how many pagelinks to show per side not counting First and Last.
March 15, 2009: 1.1.4 - More variable type checks. Looping now involves a 4D array. Code Viewing Section repaired.
March 19, 2009: 1.1.5 - TIE now shows errors if the wrong parameter type has been passed and it can not be converted. The idea of sections were added.
March 30, 2009: 1.1.6 - Changed how variables are transferred between templates. Multiple translations added. A few functions were removed / privatized. Matching symbols controlled by config. parse* other than parseConditional and parseLoop have been merged into one. Those functions are escaped by a parameter involving a hands off section.
April 1, 2009: 1.1.7 - parse and replace bug fix. Parse function's $output moved from parameter 1 to parameter 4.
April 5, 2009: 1.1.8 - Changes made to admin area. Multiple files of code per template allowed. Caching system removed.
May 6, 2009: 1.1.9 - Cleanup. Any filename that is allowed by the filesystem is allowed in the filenames.
May 9, 2009: 1.2.0 - Fixed Adding.
May 29, 2009: 1.2.1 - Optimizations. Escaping System changed. Replace function can use 3D arrays. PCRE converted to manual parsing. Fixed More errors.
May 31, 2009: 1.2.2 - Optional Config Parameters Added. ..\ disabled in template and glue calls.
May 31, 2009: 1.2.3 - Optional Config Parameters used only when the parameters are literally false.
June 3, 2009: 1.2.4 - Error handling.
June 7, 2009: 1.2.5 - Fixed bugs in Admin Area and languages. Removed magic_quotes / register_globals disabling.
June 28, 2009: 1.2.6 - Major Cleanup. Subdirectories. Readded magic_quotes_gpc prevention. Navigation limit now in a config / optional parameter. In content commenting.
July 6, 2009: 1.2.7 - Bug fixes. Parsing functionality changed.
July 7, 2009: 1.2.8 - Recursive parsing fixed.
July 10, 2009: 1.2.9 - More bug fixes. Errors reformatted. Replace function runs much faster.

SUIT
August 30, 2009: 1.3.0 - Parsing rewrite for sane recursing. Parts of TIE are now a part of SUIT. Versions are split.
August 31, 2009: 1.3.1 - The content, code, and glue directories are now defined individually.
September 4, 2009: 1.3.2 - Rewrites for efficiency in the explodeUnescape and escape functions. Escape's syntax has changed.
September 12, 2009: 1.3.3 - Rewrites for efficiency in the parse function regarding removing trees. As a result, some template syntax errors that would normally break the entire page do not. parse and getSection's parameters have been changed.
November 7, 2009: 1.3.4, Revision 5 - Nodes use functions instead of eval. Many parameters reformatted. Replace removed. Preparsing added. Temporary caching added. Comments added. Class split up into multiple files. Runs much faster.
Pending: 1.3.5 - Glue has been removed. Parser is much more flexible. Concept of attributes added.

PySUIT
September 7, 2009: 0.0.0 - Initial Release. Modeled after SUIT 1.3.2.
September 12, 2009: 0.0.1 - Modeled after SUIT 1.3.3.
November 7, 2009: 0.0.2, Revision 11 - Modeled after SUIT 1.3.4.

TIE
August 30, 2009: 1.3.0 - Debug System. Mass find and replace for Admin area. Navigation now a part of TIE. Versions are split.
August 31, 2009: 1.3.1 - Updated to accomodate SUIT 1.3.1.
September 4, 2009: 1.3.2 - Updated to accomodate SUIT 1.3.2.
September 12, 2009: 1.3.3 - Updated to accomodate SUIT 1.3.3.
November 7, 2009: 1.3.4, Revision 12 - Updated to accomodate SUIT 1.3.4. SUIT is now packaged with TIE for easy setup.
Pending: 1.3.5 - Overwrite overview. mod_rewrite. MySQL support. Installer
